## 📌 Overview

**Advanced Bank Project** is a comprehensive banking management system developed in **C++** using **Object-Oriented Programming (OOP)** principles.  
It provides an advanced Command-Line Interface (CLI) to simulate real banking operations such as account management, currency handling, and financial transactions, with strict application of **Encapsulation**, **Inheritance**, **Polymorphism**, and **Abstraction**.

---

## ✨ Key Features

| Feature | Description |
|---------|-------------|
| 👤 **Client Management** | Add, delete, update, and search for clients |
| 💰 **Account Management** | Create different types of bank accounts (Current, Savings) |
| 💱 **Currency Management** | Support for multiple currencies with conversion capabilities |
| 🏧 **Financial Transactions** | Deposit, withdraw, and transfer between accounts |
| 📊 **Transaction Log** | Record all operations with display and search capabilities |
| 🔐 **Authentication System** | Secure login with password and different permission levels |
| 📋 **Comprehensive Reports** | View account statements and financial reports |

## 🚀 How to Use

### **Prerequisites**
- **Visual Studio 2022** (or any C++17 compatible compiler)
- **Git** (for cloning and contributing)

### **Setup Instructions**
1. **Clone the repository**
   bash
   git clone https://github.com/markmoheb55555-cloud/Advanced-Bank-Project---OOP---Cpp.git
   

2. **Open the project**
   - Navigate to the project folder
   - Double-click the `OOP Project 5.sln` file

3. **Run the project**
   - Press `Ctrl + F5` or select **Debug → Start Without Debugging**

4. **Login**
   - Use default credentials (if available) or register as a new user

## 🎯 OOP Concepts Applied

| Concept | Implementation in Project |
|---------|---------------------------|
| **Encapsulation** | Using `private` and `public` access specifiers to control data access |
| **Inheritance** | Deriving specialized classes from base classes |
| **Polymorphism** | Using virtual functions for runtime polymorphism |
| **Abstraction** | Clean interfaces to hide internal implementation details |
| **Static Members** | Shared functions and variables across all objects |
